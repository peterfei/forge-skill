# 方法论提炼框架

> 如何从海量信息中提炼出一个方法论的核心结构，并确保它不是泛泛建议而是可执行的工具。

---

## 核心问题

不是所有"听起来有道理的观点"都是方法论。提炼方法论面临三个挑战：

1. **信号 vs 噪音**：一个方法论有成百上千的描述，哪些才是核心？
2. **操作化**：知道原理 ≠ 知道怎么做。如何把抽象原理转化为 step-by-step 操作协议？
3. **边界**：方法论的局限比优势更重要。如何诚实地标注边界？

---

## 三重验证标准

提炼出的每个候选"核心原理"必须通过三重验证：

| 验证维度 | 标准 | 通过条件 | 失败处理 |
|---------|------|---------|---------|
| **跨域复现** | 在 ≥2 个不同领域/行业中出现 | 有来自不同领域的实际应用证据 | 降级为操作规则 |
| **生成力** | 能推断对新问题的分析方向 | 用该原理可以推导出对新问题的分析路径 | 降级为操作规则 |
| **排他性** | 不是所有方法论都这样思考 | 至少存在一个方法论明确不采用这种方式 | 丢弃 |

**判定规则**：
- 三重全通过 → **核心原理**（纳入 Skill 的核心原理章节）
- 通过 1-2 重 → **操作规则**（纳入操作协议的具体步骤中）
- 0 重 → **丢弃**

### 验证示例

**候选观点**："还原到基本事实"

| 维度 | 验证 | 结果 |
|------|------|------|
| 跨域复现 | SpaceX（航天）用物理极限推导火箭成本；Tesla（汽车）用材料成本推导电池价格 | 通过（≥2 域） |
| 生成力 | 面对定价问题 → 回到"价值交换的基本事实"而非类比竞品 | 通过 |
| 排他性 | 系统思维不需要回到最底层，它关注的是要素之间的关系 | 通过 |
| **判定** | **核心原理** | |

**候选观点**："用物理极限做约束"

| 维度 | 验证 | 结果 |
|------|------|------|
| 跨域复现 | 仅在 Musk 的 SpaceX 案例中明确看到 | 不通过（仅 1 域） |
| 生成力 | 对非物理问题指导有限 | 边缘 |
| 排他性 | 许多工程方法都考虑约束 | 不通过 |
| **判定** | **操作规则**（纳入步骤的具体提示） | |

## 五层结构提取

每个方法论必须提取以下五层结构：

### 第 1 层：方法框架（核心原理）

- 数量：3-7 个核心原理
- 格式：名称 + 一句话定义 + 跨域证据 + 局限
- 约束：每个核心原理须附 ≥2 个跨域证据

### 第 2 层：操作流程（Agentic Protocol）

每个方法论必须有可执行的 step-by-step 流程：

```
步骤 1: [动作] — [目的]
  输入: [需要什么]
  输出: [产出什么]
  检查点: [怎么判断步骤完成]

步骤 2: [动作] — [目的]
  ...
```

**Agentic Protocol 三步结构**：
- **Step 1: 问题分类** — 判断该方法是否适用于当前问题
- **Step 2: [方法名]式分析** — 研究维度须从核心原理推导，禁止使用通用"搜索相关信息"
- **Step 3: [方法名]式输出** — 基于分析结果的格式化输出

关键约束：Step 2 的研究维度必须**从核心原理自动推导**，不能是通用模板。

### 第 3 层：适用边界

| 维度 | 必须回答 |
|------|---------|
| 最适用场景 | 该方法论表现最好的问题类型 |
| 不适用场景 | 明确不该使用该方法论的情况 |
| 常见误用模式 | 人们最容易犯的错误 |
| 与其他方法的关系 | 互补/互斥/可替代 |
| 失效信号 | 使用过程中发现方法失效的信号 |

### 第 4 层：应用模式

- 典型成功案例（2-3 个）
- 典型失败案例（1-2 个）
- 跨域迁移案例（1-2 个）

### 第 5 层：诚实边界

必须明确标注：
- 方法论本身的理论局限
- 实践中常见的执行偏差
- 信息截止日期
- 哪些维度信息不足
- 禁止使用"不能替代专业建议"类空话

## 信息源管理

### 信息源优先级

| 来源类型 | 权重 | 说明 |
|---------|------|------|
| 方法论创始人的原始著作 | 最高 | 定义了方法的本质 |
| 同行评议的学术论文 | 高 | 经过验证的知识 |
| 成功实践案例记录 | 高 | 证明方法有效 |
| 批判性分析和反面案例 | 高 | 揭示方法局限 |
| 教科书/综述文章 | 中 | 系统化但可能简化 |
| 博客/社交媒体讨论 | 低 | 可能有洞察但需验证 |

### 信息源黑名单

- 知乎（洗稿严重）
- 微信公众号（封闭生态）
- 百度百科等低质量百科

### 一手来源占比

最终 Skill 中引用的一手来源（创始人著作、同行评议论文、原始案例记录）必须 >50%。
