# 国际化贡献指南

> forge-skill 从第一天起就是多语言架构。本文档指导社区成员如何贡献翻译和本地化内容。

---

## 核心原则

1. **核心框架跨语言一致**：核心原理、操作协议、适用边界、误用检测规则、诚实边界在不同语言版本中完全一致（仅语言不同）
2. **案例本地化**：不同语言版本使用当地商业/文化案例
3. **术语保留原文**：关键术语在翻译后保留原文括号注释
4. **知识源优先级**：原始语言文献 > 英语翻译/分析 > 其他语言翻译

---

## 贡献类型

### 类型 1: 术语映射贡献（i18n/term-glossary/）

为 `i18n/term-glossary/` 目录下的 YAML 文件贡献新条目或新语言。

**数据结构**：
```yaml
# i18n/term-glossary/ja.yaml（日语示例）
first_principles:
  name: 第一原理思考
  aliases: [第一原理, ファースト・プリンシプル]
  origin: Aristotle
  key_concepts:
    - term: 基本事実
      en: "fundamental truths"
    - term: 物理的限界
      en: "physical limits"
```

**质量要求**：
- 术语翻译需经母语者审核
- `aliases` 至少包含 2 个常用别名
- `key_concepts` 需要翻译该方法论的所有核心概念

**PR Review 要求**：至少 1 名母语者审核通过

### 类型 2: 方法论 Skill 翻译

将已有方法论 Skill 翻译为另一种语言版本。

**一致性检查清单**：
- [ ] 核心原理数量和定义与基准版一致
- [ ] 操作协议步骤与基准版一致
- [ ] 适用/不适用判断标准与基准版一致
- [ ] 误用检测规则与基准版一致
- [ ] 诚实边界与基准版一致
- [ ] 案例已替换为本地化案例
- [ ] 触发词已替换为本地语言
- [ ] 关键术语保留原文括号注释

**PR Review 要求**：至少 1 名母语者审核通过

### 类型 3: 文化背景补充

为已有 Skill 补充本地文化背景说明。

**示例**：
```
第一性原理源自 Aristotle 的物理学思考，在中国文化中，
王阳明的'知行合一'和'格物致知'有类似但不同的思想内核。
注意不要混淆——格物致知强调从外部事物中获取知识，
第一性原理强调从基本假设出发推导。
```

**质量要求**：
- 明确区分"类似但不同"的关系
- 避免强行关联无关的文化概念
- 标注信息来源
